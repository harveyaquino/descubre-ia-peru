# Descubre tu primer empleado IA

Experiencia conversacional para emprendedores y PYMEs del Perú: "Lucía", una asistente de IA, entrevista al visitante unos minutos, **diagnostica en qué momento está su negocio** (Base, Sistemas, Contenido o Escalar) y le entrega una **ficha personalizada** con una herramienta estrella + un **plan de 7 días**.

Las recomendaciones salen **solo del catálogo del método** "IA para tu negocio: de la base a la automatización" (ver `IA_para_tu_negocio.pdf`): ChatGPT, Looka, Khroma, Durable, Notion AI, Desygner, Canva, Riverside, Ocoya, Fireflies, Perplexity, Tidio, HubSpot Breeze, Claude y Zapier.

Construida con Next.js, Supabase y Vercel; la conversación la genera Claude (`claude-opus-4-8`).

## 🚀 Stack Tecnológico

- **Frontend**: Next.js 14 + React 18
- **Backend**: Supabase (PostgreSQL)
- **Hosting**: Vercel
- **Versionado**: GitHub

## 📋 Configuración Inicial

### 1. Clonar el Repositorio

```bash
git clone https://github.com/tu-usuario/ia-peru-taller.git
cd ia-peru-taller
```

### 2. Instalar Dependencias

```bash
npm install
```

### 3. Configurar Supabase

1. Crear una cuenta en [Supabase](https://supabase.com)
2. Crear un nuevo proyecto
3. En la sección **SQL Editor**, ejecutar:

```sql
CREATE TABLE leads (
  id BIGINT PRIMARY KEY GENERATED BY DEFAULT AS IDENTITY,
  email TEXT NOT NULL,
  ficha JSONB,
  created_at TIMESTAMPTZ DEFAULT NOW()
);

CREATE INDEX idx_leads_created_at ON leads(created_at);
```

> El lead solo se guarda si el visitante decide, al final, dejar su correo
> para recibir su ficha. La columna `ficha` guarda el JSON completo del
> resultado (empleado IA recomendado + plan de 7 días).

4. Obtener tus credenciales:
   - Ir a **Project Settings** > **API**
   - Copiar `Project URL` y la `service_role` key (no la anon)

### 4. Configurar Variables de Entorno

Copia `.env.local.example` a `.env.local` y rellena las 3 claves:

```bash
NEXT_PUBLIC_SUPABASE_URL=https://tu-proyecto.supabase.co
SUPABASE_SERVICE_ROLE_KEY=tu_service_role_key   # Settings > API (SECRETA)
ANTHROPIC_API_KEY=sk-ant-xxxxx                  # console.anthropic.com (SECRETA)
```

> **Importante:** la conversación se procesa en el servidor (`pages/api/chat.js`),
> así las claves nunca llegan al navegador. La `service_role` key y la
> `ANTHROPIC_API_KEY` jamás deben usar el prefijo `NEXT_PUBLIC_`.

### Cómo funciona el flujo

1. El visitante conversa con Lucía (`POST /api/chat`), que hace 4-6 preguntas
   sobre su negocio. Sin registro.
2. Cuando tiene contexto suficiente, Claude (`claude-opus-4-8`) llama a una
   herramienta que devuelve la ficha estructurada (empleado IA + plan de 7 días).
3. Se muestra la ficha al instante. Al final, el visitante puede dejar su correo
   (opcional, `POST /api/lead`) para recibirla.

Si Anthropic no está configurado, el chat muestra un mensaje de disculpa.
Si Supabase no está configurado, la captura de correo confirma sin guardar
(no rompe la experiencia).

### 5. Ejecutar en Desarrollo

```bash
npm run dev
```

Visitar `http://localhost:3000`
